# NaonaoVault 项目说明书

## 1. 项目位置

- 项目根目录：`x:\超级个体\密码管理工具\secure-vault`

## 2. 这是一个什么项目

NaonaoVault 是一个本地密码管理工具，技术栈是：

- Electron 29
- React 18
- TypeScript
- SQLite `better-sqlite3`
- Zustand

核心特点：

- 敏感字段只在主进程加解密
- 使用 Node `crypto` 的 `AES-256-GCM`
- 主密码使用 `PBKDF2-SHA256` 派生密钥
- 支持 AI 设置、智能录入、AI 助手
- 支持 `.svlt` 加密备份导入导出
- 支持 Git 同步

## 3. 目录说明

最重要的目录和文件如下：

- `src/main`
  - Electron 主进程代码
  - `index.ts`：主进程入口、窗口创建、IPC 注册
  - `preload.ts`：暴露给渲染层的安全 API
  - `vault-service.ts`：保险库核心服务，负责解锁、记录 CRUD、迁移
  - `crypto.ts`：加密、解密、主密码校验、密码生成
  - `database.ts`：SQLite 数据库读写
  - `backup-service.ts`：`.svlt` 备份导出导入
  - `git-sync-service.ts`：Git pull / commit / push 同步
  - `ai-service.ts`：AI 测试连接、智能录入、助手问答
  - `contracts.ts`：主进程侧数据类型
- `src/renderer`
  - React 界面代码
  - `App.tsx`：应用入口和初始化流程
  - `store/index.ts`：Zustand 状态管理
  - `components/Header.tsx`：顶部栏，包含搜索、AI、设置等入口
  - `components/RecordModal.tsx`：记录新增/编辑弹窗
  - `components/SettingsModal.tsx`：设置、备份、同步、AI 配置
  - `components/SmartCaptureModal.tsx`：智能录入弹窗
  - `components/AssistantModal.tsx`：AI 助手弹窗
- `scripts`
  - `run-electron-builder.cjs`：统一调用 `electron-builder`
  - `build-clean-portable.cjs`：生成干净便携包目录
- `dist`
  - 编译输出
  - `dist/main`：主进程编译结果
  - `dist/renderer`：渲染层编译结果
- `release`
  - 打包结果输出目录

## 4. 开发和构建

### 4.1 安装依赖

在项目根目录执行：

```powershell
secure-vault
npm install
```

### 4.2 本地开发

你当前项目里最常用的是先编译再启动，或者分别启动：

```powershell
npm run dev:main
npm run dev:renderer
npm start
```

说明：

- `dev:main`：监听编译主进程 TypeScript
- `dev:renderer`：启动 Vite 开发服务器
- `start`：启动 Electron

### 4.3 构建

```powershell
npm run build
```

会生成：

- `dist/main`
- `dist/renderer`

### 4.4 测试

```powershell
npm test
```

当前已覆盖：

- `crypto`
- `backup-service`
- `database`
- `vault-service`
- `git-sync-service`
- `ai-service`

## 5. 怎么打包

### 5.1 生成安装包

```powershell
npm run dist
```

打包配置文件在：

- `electron-builder.yml`

生成结果在：

- 安装包：`release/NaonaoVault-1.0.0-setup-x64.exe`

这是给普通用户安装使用的版本。

### 5.2 生成免安装目录

```powershell
npm run pack
```

生成结果在：

- 免安装目录：`release/win-unpacked`
- 主程序：`release/win-unpacked/NaonaoVault.exe`

注意：

- 不能只发 `NaonaoVault.exe`
- 必须把整个 `win-unpacked` 文件夹一起发给别人

### 5.3 生成干净便携包

这个版本不会读取你机器上 `%APPDATA%` 的旧数据，适合测试和发别人直接解压运行。

先执行：

```powershell
npm run pack
npm run portable:clean
```

生成结果在：

- 目录：`release/NaonaoVault-1.0.0-clean-portable`
- 启动脚本：`release/NaonaoVault-1.0.0-clean-portable/NaonaoVault-Clean.cmd`

如果要压缩成 zip 再发别人：

```powershell
Compress-Archive -Path .\release\NaonaoVault-1.0.0-clean-portable\* -DestinationPath .\release\NaonaoVault-1.0.0-clean-portable.zip -Force
```

当前已经存在的压缩包在：

- `release/NaonaoVault-1.0.0-clean-portable.zip`

## 6. 我该发给别人哪个文件

有三种发法：

### 6.1 给普通用户

发这个安装包：

- `release/NaonaoVault-1.0.0-setup-x64.exe`

### 6.2 给测试人员

发这个 zip：

- `release/NaonaoVault-1.0.0-clean-portable.zip`

对方解压后，运行：

- `NaonaoVault-Clean.cmd`

不要直接双击 `NaonaoVault.exe`，因为干净模式依赖这个启动脚本来隔离数据目录。

### 6.3 给自己本机临时跑

直接打开：

- `release/win-unpacked/NaonaoVault.exe`

如果你不想让它读本机旧数据，就不要直接用这个，而是用干净便携包。

## 7. 首次使用怎么操作

首次打开应用时：

1. 如果本机没有历史数据，会看到初始化界面
2. 输入主密码
3. 确认后创建保险库
4. 进入主界面后即可新增记录

常用入口：

- 顶部搜索框：搜索记录
- 顶部 AI 按钮：打开 AI 助手
- 顶部智能录入按钮：从文本自动生成记录草稿
- 顶部设置按钮：打开 AI、备份、同步、安全设置
- 顶部锁定按钮：手动锁定保险库

快捷键：

- `Ctrl/Cmd + F`：聚焦搜索
- `Ctrl/Cmd + N`：新建记录
- `Ctrl/Cmd + L`：锁定保险库
- `Ctrl/Cmd + ,`：打开设置

## 8. 数据保存在哪里

### 8.1 安装版和普通运行

默认数据目录：

- `%APPDATA%\secure-vault`

数据库文件：

- `%APPDATA%\secure-vault\secure-vault.db`

### 8.2 干净便携包

数据保存在包内目录：

- `release/NaonaoVault-1.0.0-clean-portable/user-data`

这意味着：

- 便携包不会污染你系统原来的 `%APPDATA%`
- 删除 `user-data` 目录，就相当于把这个便携包恢复到全新状态

## 9. 如何清理旧数据

如果本机之前跑过旧版本，可能残留脏数据。

正常需要清理的是：

- `%APPDATA%\secure-vault`

如果你只想临时测试，不想动系统数据，直接用干净便携包即可。

## 10. 安全说明

当前实现的安全边界如下：

- 渲染层提交明文表单
- 主进程负责加密入库
- 主进程负责解密返回 UI
- AI API Key 在本地以加密形式保存
- 备份文件 `.svlt` 也是加密快照，不是明文数据库

当前加密相关实现位置：

- `src/main/crypto.ts`
- `src/main/vault-service.ts`
- `src/main/database.ts`

## 11. AI 怎么配置

打开应用后：

1. 点击右上角“设置”
2. 找到“AI 设置”
3. 填三个字段

字段说明：

- `Base URL`
  - 你的模型服务地址
  - 要求兼容 OpenAI 风格的 `chat/completions` 接口
  - 常见写法是带 `/v1`
- `Model`
  - 你实际可用的模型名
- `API Key`
  - 你的服务密钥

保存后可以点“测试连接”。

AI 能力说明：

- 智能录入
  - 先本地提取密码、Key、账号、网址
  - 发给模型的是脱敏结构，不直接发原始敏感值
- AI 助手
  - 先本地检索候选记录
  - 模型只处理候选摘要，不直接拿到密码或 Key
  - 真正显示密码/Key 仍需本地显式触发

代码位置：

- 配置界面：`src/renderer/components/SettingsModal.tsx`
- AI 服务：`src/main/ai-service.ts`

## 12. 备份怎么用

在“设置与同步”里可以操作：

- 导出备份
- 导入备份

备份格式：

- 文件扩展名：`.svlt`

说明：

- 备份是加密快照
- 导入时不是粗暴覆盖，而是走 merge
- 同一 vault 内采用 last-write-wins + tombstone 规则

代码位置：

- `src/main/backup-service.ts`

## 13. Git 同步怎么配置

先决条件：

- 电脑上已经安装 Git
- Git 命令在系统 PATH 中可用
- 你本机已经具备仓库访问权限

在“设置与同步”里填写：

- 远程仓库地址
- 分支
- 本地工作目录
- 快照文件名

同步逻辑：

1. 初始化或检查本地仓库
2. `git pull --rebase`
3. 读取仓库中的 `.svlt`
4. 与本地保险库合并
5. 生成新的 `.svlt`
6. `git add`
7. `git commit`
8. `git push`

代码位置：

- `src/main/git-sync-service.ts`

注意：

- Git 同步同步的是加密快照，不是 SQLite 数据库文件
- 如果 `git` 未安装、认证失败、远端仓库不可访问，会在设置面板里显示错误

## 14. 常见问题

### 14.1 为什么一打开就让我输密码

通常有两种原因：

- 本机 `%APPDATA%\secure-vault` 里已经有旧数据
- 你直接运行了读取系统数据目录的版本

解决方式：

- 清理 `%APPDATA%\secure-vault`
- 或直接使用 `NaonaoVault-1.0.0-clean-portable.zip`

### 14.2 为什么干净便携包不能直接双击 `NaonaoVault.exe`

因为干净模式依赖：

- `NaonaoVault-Clean.cmd`

它会强制把数据目录切到包内的 `user-data`。

### 14.3 Windows 弹出安全提示怎么办

当前安装包没有做代码签名，所以 Windows 可能弹 SmartScreen 提示。

这不是包损坏，而是未签名软件的正常行为。

### 14.4 忘了主密码怎么办

当前设计下，主密码不会明文保存，不能直接反查。

如果只是测试环境，需要干净状态：

- 用干净便携包
- 或删除对应数据目录重新初始化
